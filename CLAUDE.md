# Instructions for Claude working in this repo

- After making any change that affects the pipeline architecture, model
  configuration (`.env`), file structure, or known issues, update
  `STATUS.md` to reflect it — add a line to its "Recent change log" and
  update the relevant section (model config table, key files, known issues).
  Do this in the same turn as the change, not as a separate follow-up.
- Keep `STATUS.md` short and current, not a full history — trim or
  consolidate older change-log entries if the file starts sprawling.
